Building a SAPP container

The process of building a SAPP container is simply a matter of directing Singularity to build a particular specfile. You can increase verbosity and/or debugging by using the options --verbose and --debug.

Documentation for the specfile can be found here. 

$ singularity build
USAGE: singularity (options) build [SAPP specfile] (SAPP target)

$ singularity build /path/to/singularity/sapp.sspec
Building 'example'
WROTE: example.sapp

The output file example.sapp is marked executable and can be executed on any Linux computer where Singularity is installed.
