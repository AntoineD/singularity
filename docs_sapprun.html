Executing a SAPP container

There are two mechanisms for launching a SAPP container. The first you can execute the SAPP file directly or you can launch the SAPP through the Singularity run command. In either case, all arguments passed to the container will be passed through Singularity and directly given to the container's runscript driver (%runscript).

Running a SAPP container directly

Each SAPP file is built with a Singularity wrapper as an intrepeter so when executed the shell understands that Singularity should be used to execute this application. For most cases, there is no need to run the full Singularity command just to execute a SAPP container, but it does give you debugging and possibly future feature options above and beyond the default.

It is important to note that while the SAPP file is executable, it *requires* Singularity to be installed for it to be able to run!

Using Singularity to run a SAPP container

You can also use Singularity itself to launch the container. Benefits to doing this are additional Singularity options and features (e.g. --debug). This should be done as follows: 

$ singularity run /path/to/singularity/file.sapp [arguments]
