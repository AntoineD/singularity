Singularity is a package building and run-time environment built to achieve portable application bundles which can be then run on any Linux system without ever having to worry about portability or dependencies. These portable Singularity application bundles never need to be installed, and can be executed directly at run-time!

About Singularity

Background

There are several methods to implement portable application stacks at present. Among them are containers and virtual machines. But when using these solutions to package a single application (or application stack), one ends up having to manage the entire operating system overhead as well (this includes security, updates, the size overhead, etc..) just to distribute a single application or work flow. While virtual machines and containers have very strong usage benefits, application portability and distribution is not an efficient use of these technologies.

Singularity works on a completely different premise. Contrary to building and managing an entire operating system environment to distribute a custom application, configuration or demonstration, Singularity packages all of the required dependencies and creates an executable package that can be distributed easily, safely, and reliably.

Portability

A Singularity Application (SAPP) is a package bundle which contains within it all of the required dependencies necessary to run. It is like having a statically compiled binary which can be distributed easily from system to system without concern of even the C library compatibility. The SAPP can contain anything from a simple binary executable to an entire pipeline or work flow of multiple applications. There could be multiple services which can be run within the SAPP environment and interaction with the SAPP file is fully configurable. For example in the simple configuration you may choose to emulate the program arguments so it operates just like running the application directly.

Reproducibility

Each SAPP includes all of the application's necessary run-time libraries and can even include the required data and files for a particular application to run. This encapsulation of the entire user-space environment facilitates not only portability but also reproducibility.
